## Why this foundation matters

A personal site looks small from the outside, but it still needs a durable system underneath it. Portfolio entries, essays, metadata, motion, and responsive behavior all start to overlap quickly.

### The emotional target

The site should feel calm and technical rather than loud. That meant prioritizing whitespace, typography, and restrained accent color before adding any flourish.

## The stack choices

I wanted a stack that made content editing easy while preserving modern frontend patterns.

### What made the cut

- Next.js App Router for server-first page composition
- Tailwind with CSS custom properties for design tokens
- MDX for long-form content with embedded components

```ts
export const foundation = {
  shell: "header-footer-layout",
  content: "mdx-plus-json",
  theme: "class-based-dark-mode",
};
```

<Callout tone="info" title="A practical rule">
If a decision improved both authoring and presentation, it stayed. If it only made the implementation feel clever, it usually got cut.
</Callout>

## Design tokens before components

The fastest way to keep pages coherent was to define the shared language first: colors, spacing, container widths, shadows, and motion timing.

### A token mindset

| Token family | Why it mattered |
| --- | --- |
| Color | Establish contrast and calm accent use |
| Spacing | Keep rhythm consistent across every page |
| Motion | Avoid one-off animation behavior |

> Strong foundations make later page work faster, calmer, and easier to reason about.

## What this unlocked

The result is a site that can grow without feeling stitched together. New pages inherit the same visual rules, metadata defaults, and long-form content behaviors by default.
